Глава 10. Резервное копирование 10.1. Выгрузка данных В наборе утилит, прилагающихся к любой СУБД, обязательно есть инструмен- ты для выгрузки и загрузки данных. Как правило, данные сохраняются либо в тек- стовом виде, либо в двоичном формате, специфичном для конкретной платфор- мы. В таблице ниже приведён список таких инструментов: Двоичный формат Текстовый формат DataPump Export SQL*Plus Oracle и DataPump Import; и SQL*Loader Export и Import PostgreSQL pg_dump/pg_dumpall pg_dump/pg_dumpall и pg_restore и psql Microsoft SQL Server bcp bcp Db2 unload и load unload и load MySQL mysqldump/mysqlpump и mysql/mysqlimport MongoDB mongodump mongoexport и mongorestore и mongoimport Cassandra nodetool snapshot cqlsh и sstableloader Текстовый формат хорош тем, что его можно редактировать или даже созда- вать внешними программами. Двоичный, в свою, очередь позволяет быстрее вы- гружать и загружать данные за счёт распараллеливания загрузки и экономии ре- сурсов процессора на преобразовании форматов. Несмотря на простоту и очевидность идеи выгрузки данных, для резервирова- ния нагруженных промышленных баз такой метод применяют редко. Вот причи- ны, по которым выгрузка не подходит для полноценного резервного копирования: • процесс выгрузки создаёт значительную нагрузку на систему-источник; • сделать согласованную выгрузку всей базы данных при высокой нагрузке практически невозможно, поскольку СУБД вынуждена хранить снимок сво- его состояния на момент начала выгрузки.
