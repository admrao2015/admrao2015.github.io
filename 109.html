Глава 3. Структуры хранения данных При вычислении агрегатных функций (sum, avg и других) можно использо- вать эти данные непосредственно, без распаковки. • Битовый вектор (bit-vector). Этот метод кодирования похож на битовые индексы в Oracle и используется для колонок, содержащих малое количество различных значений. В файле данных помещается список значений, а затем для каждого значения создаётся битовая карта, где единицей отмечены по- зиции, в которых встречается это значение: A E A B A E B B {A, 0xA8}; {B, 0x13}; {E, 0x44} A 1 0 1 0 0 0 0 1 0xA 0x8 • Словарное кодирование с фиксированным размером (ﬁxed width dic- tionary). Для колонок с повторяющимися значениями создаётся словарь, и значения заменяются на позицию в словаре: Москва; С.-Петербург; Казань 00 00 00 01 00 00 10 01 00 Москва С.-Петербург Москва Москва Москва Москва Казань С.-Петербург Москва Код Хаффмана более компактен (на рисунке выше значение «Москва» с точ- ки зрения объёма хранения было бы выгоднее закодировать одним битом), но нередко компактность приносится в жертву фиксированному размеру. • Подвижная система отсчёта (frame-of-reference). Если значения в колон- ке близки, система сохраняет базовое значение («точку отсчёта») и разницу текущего значения с базовым: 1524375: 0,1,2,5,4,7,8 1524375 1524376 1524377 1524379 1524382 1524383 1524380 • Вариант подвижной системы отсчёта — разностное кодирование (delta coding), когда вместо числа сохраняется его разница с предыдущим числом: 108
